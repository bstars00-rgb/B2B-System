## 2. 데모 확인 방법 (기획팀용 가이드)

1. 데모 URL 접속 → 대문에서 **약관 동의 체크 후 Sign In** (자격증명은 미리 채워져 있음 — 실제 인증 없음)
2. **보라색 `UP` 배지 = 닷비즈 원본과 다른 고도화 지점** — 마우스를 올리면 무엇이 다른지 설명이 뜹니다. 헤더의 **✨ 버튼**으로 배지를 숨기면 순수 클론 화면이 됩니다. (원본에 있던 기능을 복원한 것에는 배지가 없습니다)
3. 언어: 헤더 **🌐 셀렉트**가 전역 설정 — Playbook·FAQ가 함께 전환됩니다 (일/베/중은 번역 전이라 영어 폴백 + 안내)
4. 추천 시연 동선: 대문(광고판 로테이션·약관 모달) → Create Booking에서 "osaka" 검색(로딩→26곳·필터·사진) → Select(새 탭 룸리스트) → Rooms 2로 재검색(요금 배수) → 요금 Select → 예약 모달(아동 생년월일 검증) → Create → 원래 탭 Bookings 반영 확인

## 3. 이번에 구현·검증된 것 (화면별)

### 3.1 대문(로그인) — 신규 기획 1호 구현

- **광고판**: 좌측 절반을 "우리는 누구인가/무엇을 하는가" 캠페인 영역으로 — 3종 캠페인이 자동 로테이션(8초)되며, 소재 파일에 추가만 하면 교체되는 **주기 운영 구조**. 대형 타이포(68px) + AI 시대 무드 배경(뉴럴 네트워크·펄스·글로우, "AI Rate Engine — Online" 상태 표시)
- **트렌드형 로그인**: DOTBIZ 브랜딩, 비밀번호 표시 토글, Remember me, 다크모드 토글, Create Account
- **약관 외부 열람**: Terms & Condition **전문 26개조(2026-03-14 시행)** · Privacy Policy 전문을 로그인 전 모달로 열람, 동의 없이는 로그인 불가. 포털 푸터에도 동일 링크

### 3.2 호텔 검색·예약 흐름 (Create Booking → 룸리스트 → 예약)

- 지역 검색 시 실호텔 목록(오사카 26곳 실코드), 결과 기반 필터(성급·숙소유형·체인 브랜드)·페이저
- 실사이트처럼 **조회 지연 + Searching… 로딩**, 호텔 **실사진**(목록 카드·상세 갤러리)
- Select 시 **새 탭 룸리스트**(원래 검색 결과 유지 — 다른 호텔 계속 비교 가능)
- 룸리스트에서 **검색 후 조건 변경**: 체크인/아웃·Nights·Rooms(룸별 인원·아동나이) 변경 → 재검색, 방 2개면 요금도 배수 재계산
- 입력 안전장치: 첫 진입 시 날짜 빈 상태(실사이트 동일), **달력 과거 날짜 비활성**, 체크아웃은 체크인 다음날부터, **아동 생년월일-검색 나이 불일치 경고**(체크인 기준 만 나이)
- 업무 규칙: **Unpaid 예약은 Voucher 발행 불가**(실사이트 규칙 반영)
- 새 탭에서 만든 예약이 원래 탭 Bookings에 **실시간 반영**(새로고침에도 유지)

### 3.3 Bookings (예약 관리)

- 행 **체크박스 실작동**(전체선택/개별선택), **엑셀식 컬럼 폭 드래그 조절** — 실사이트 그리드와 동일 조작감

### 3.4 매뉴얼·게시판·언어

- **Ellis Playbook 내장**(원본에 없는 고도화): 6챕터 문서형 매뉴얼, 포털 언어 설정 연동
- FAQ·Playbook **언어팩(영/한)** — 언어는 전역 설정 한 곳에서만 변경(원칙), 일/베/중 번역은 콘텐츠팀 진행 중
- 사이드바 **메뉴 검색 실작동**(한/영 키워드·하이라이트)

### 3.5 참고 — AI 요금 검색(트랙 A) 개선

- 과거 날짜 임의 단정 금지(다음 도래 연도 해석+가정 명시), 목적지 없으면 되묻기, **"더블+트윈 각각 1개씩" 룸타입 분리 인식** — 상세는 기획서 ①

> 25건 전체의 항목별 커밋 이력은 [상세 명세 §3.1](spec-b-dotbiz-enhancement.md) 참조.

## 5. 기획팀 의사결정 요청 (Open Questions)

1. **대문 캠페인 소재** — 제작 주체(마케팅/콘텐츠)와 교체 주기(제안: 격주) 확정
2. **더블·트윈 분리 예약의 예약 단위** — ELLIS 코드 1건+룸 2행인지, 룸당 코드 발번인지 실사이트/ELLIS 확인 필요
3. **오피포인트 정책** — 적립률·사용처·세무(리베이트 이슈) 법무/재무 검토
4. **아동 나이 검증 강도** — 현재 체크인 기준 만 나이 엄격 경고. 한국식 나이 입력 관행 고려 ±1세 허용/보정 제안 버튼 등 대안 있음
5. **물량 전환 측정** — API 경유 vs 닷비즈 직접 예약을 구분 집계할 내부 데이터 소스 확보
