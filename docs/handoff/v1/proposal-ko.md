## 1. 한 장 요약

| 항목 | 내용 |
|------|------|
| 목적 | ① 현장 OP의 불편을 제거해 **고객 친화 플랫폼**으로 전환 ② API 고객 물량이 대부분인 상황에서 **닷비즈 고도화로 닷비즈 고객을 추가 확보하고 물량을 크게 늘림** (API 고객 이관이 아니라 **닷비즈 채널 자체를 키우는 것**) |
| 방식 | 실사이트(ohmyhotel.biz)를 1:1 클론한 프로토타입에서 UX를 먼저 검증하고, 검증된 안을 운영 반영 제안 |
| 1차 범위 | **완료 25건**(원본 기능 복원 = 패리티 + 신규 고도화) + **신규 기획 4건 제안**(§4) |
| 진행 원칙 | 하나의 불편 = 하나의 개선 = 당일 반영 · 실측 기반(폰트 Pretendard·브랜드 #EF7F29까지 원본과 동일) |

## 2. 데모 확인 방법 (기획팀용 가이드)

1. 데모 URL 접속 → 대문에서 **약관 동의 체크 후 Sign In** (자격증명은 미리 채워져 있음 — 실제 인증 없음)
2. **보라색 `UP` 배지 = 닷비즈 원본과 다른 고도화 지점** — 마우스를 올리면 무엇이 다른지 설명이 뜹니다. 헤더의 **✨ 버튼**으로 배지를 숨기면 순수 클론 화면이 됩니다. (원본에 있던 기능을 복원한 것에는 배지가 없습니다)
3. 언어: 헤더 **🌐 셀렉트**가 전역 설정 — Playbook·FAQ가 함께 전환됩니다 (일/베/중은 번역 전이라 영어 폴백 + 안내)
4. 추천 시연 동선: 대문(광고판 로테이션·약관 모달) → Create Booking에서 "osaka" 검색(로딩→26곳·필터·사진) → Select(새 탭 룸리스트) → Rooms 2로 재검색(요금 배수) → 요금 Select → 예약 모달(아동 생년월일 검증) → Create → 원래 탭 Bookings 반영 확인

## 3. 이번에 구현·검증된 것

완료 25건은 성격이 두 가지로 나뉩니다 — **① 고도화(닷비즈 원본에 없는 신규)** 와 **② 클론(패리티) 복원(원본 닷비즈 기능을 지시대로 재현)**. 데모의 보라 `UP` 배지는 ①에만 붙습니다.

### 3.1 고도화 — 닷비즈 원본에 없는 신규 (본 기획의 제안 사항)

- **대문(로그인) 리뉴얼 — 신규 기획 1호**
  - 광고판: 좌측 절반을 "우리는 누구인가/무엇을 하는가" 캠페인 영역으로 — 3종 캠페인 자동 로테이션(8초), 소재 파일에 추가만 하면 교체되는 **주기 운영 구조**. 대형 타이포(68px) + AI 시대 무드 배경(뉴럴 네트워크·펄스·글로우, "AI Rate Engine — Online" 상태 표시)
  - 트렌드형 로그인: DOTBIZ 브랜딩, 비밀번호 표시 토글, Remember me, 다크모드 토글, Create Account
  - 약관 외부 열람: Terms & Condition **전문 26개조(2026-03-14 시행)** · Privacy Policy 전문을 로그인 전 모달로 열람, 동의 게이트
- **Ellis Playbook 내장**: 6챕터 문서형 매뉴얼(원본은 별도 사이트), 포털 언어 설정 연동
- **아동 생년월일-검색 나이 불일치 경고**(체크인 기준 만 나이 — 원본은 입력만 받고 검증 없음)
- **UP 배지 표기 시스템**: 고도화 지점을 화면에서 식별(✨ 토글)
- (참고 — AI 요금 검색, 트랙 A): 과거 날짜 임의 단정 금지, 목적지 되묻기, "더블+트윈 각각 1개씩" 분리 인식 — 상세는 기획서 ①

### 3.2 클론(패리티) 복원 — 원본 닷비즈 기능의 재현 (클론 지시 이행분)

> 아래는 **원래 닷비즈에 있던 기능**입니다. 프로토타입에 지시대로 1:1 재현·검증을 완료했으며, 신규 제안이 아닙니다.

- **호텔 검색·예약 흐름**: 지역 검색 실호텔 목록(오사카 26곳 실코드)·필터(성급/유형/체인)·페이저, 조회 지연+Searching… 로딩, 호텔 사진, Select 시 **새 탭 룸리스트**(검색 결과 유지), 검색 후 조건 변경·다중 룸 요금 재계산, 첫 진입 날짜 빈 상태, 달력 과거 날짜 비활성, Unpaid 예약 Voucher 발행 불가, 예약 생성→Bookings 반영
- **Bookings(예약 관리)**: 행 체크박스(전체/개별 선택), 엑셀식 컬럼 폭 드래그 조절 — 원본 그리드와 동일 조작감
- **언어팩**: 원본이 보유한 5개 언어(영·한·중·베·일) 체계 복원 — 영/한 콘텐츠 완료, 언어는 전역 설정 한 곳에서만 변경, 일/베/중 번역은 콘텐츠팀 진행 중
- **사이드바 메뉴 검색**(Enter Menu name): 한/영 키워드 매칭·일치 하이라이트로 재현
- **약관·개인정보**: 원본 문서 전문 수록 + 포털 푸터 링크
- **실측 UI 동일화**: 폰트(Pretendard 웹폰트)·브랜드 오렌지(#EF7F29)·기본 12px/#333 등 원본 실측값 그대로

> 25건 전체의 항목별 커밋 이력은 [상세 명세 §3.1](../../plan/spec-b-dotbiz-enhancement.md) 참조.

## 4. 신규 기획 제안 (하반기 실행)

| 2 | 예약생성 OP 편의 3종 | ⑴ 룸타입 솔팅(트윈/더블 필터) ⑵ **더블·트윈 분리 예약**(한 흐름에 룸타입 2개 담기) ⑶ **요금·룸타입 복사**(견적 텍스트 클립보드) | 초안 완료 — 8월 상세 기획 확정 |

| 4 | 닷비즈 채널 성장 | 고도화된 포털을 세일즈 자산으로 신규 고객사 확보(다국어·셀프 온보딩) + 기존 고객 사용 확대 → 닷비즈 채널 물량 성장 · 월간 성장 지표 추적 | 8월 실행안·측정 체계 확정 |

## 5. 기획팀 의사결정 요청 (Open Questions)

1. **대문 캠페인 소재** — 제작 주체(마케팅/콘텐츠)와 교체 주기(제안: 격주) 확정
2. **더블·트윈 분리 예약의 예약 단위** — ELLIS 코드 1건+룸 2행인지, 룸당 코드 발번인지 실사이트/ELLIS 확인 필요
3. **오피포인트 정책** — 적립률·사용처·세무(리베이트 이슈) 법무/재무 검토
4. **아동 나이 검증 강도** — 현재 체크인 기준 만 나이 엄격 경고. 한국식 나이 입력 관행 고려 ±1세 허용/보정 제안 버튼 등 대안 있음
5. **닷비즈 채널 성장 측정** — 닷비즈 채널 예약 건수·거래액·신규 고객사 수를 월간 집계할 내부 데이터 소스 확보
