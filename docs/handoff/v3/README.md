## 들어오는 상태 (2026-07-17 2차 마무리 시점)

### 회신 완료 — 3차에서 설계로 옮길 것

2차 의사결정 7건 중 **6건이 확정**되어 프로토타입·명세에 반영됐다([v2 기획안 §6](../v2/proposal-en.md)). 그중 **설계 작업이 새로 생긴 것**은 하나다:

| 출처 | 확정 | 3차에서 할 일 |
|------|------|--------------|
| 2차 ② | 베스트셀러 랭킹 = **마케팅 편집물**(순위를 우리가 결정) | **랭킹 편집 수단 설계** — 배포 없이 순위를 바꾸는 관리 화면/CMS. 전월 순위 보관(MoM 산출용) · 예약 가능 불변조건 · 최저가 요금 조달. 대문 캠페인(v1 F-1.1)과 같은 부류이므로 **편집 수단을 함께 묶어 설계할지 검토** |
| 2차 ③ | 집계 = **실시간** | 대시보드 집계 **서버 이관 설계** — 실시간 질의이므로 `(seller_id, booking_date/check_in/check_out)` 인덱스, 대시보드 1회 조회를 단일 응답으로 |
| 2차 ①⑦ | 셀러는 자기 예약만 · 공급사명 미노출 | (반영 완료 — 추가 작업 없음) |
| 2차 ⑥ | Year-End 탭 존치 | (반영 완료 — 추가 작업 없음) |

### 회신 대기 — 3차 착수 전 필요

| # | 항목 | 대상 |
|---|------|------|
| 1 | **Deferred Credit 정의**(2차 ⑤) | 트레이시 확인 예정 — 현재 `확정 ∧ 미수(Unpaid)`로 구현 |
| 2 | **"Bestselling" 라벨 유지 여부** | 2차 ② 확정으로 새로 생긴 질문 — 순위를 우리가 정하면 그 라벨은 계산하지 않은 판매 사실을 주장하게 된다. 유지 vs `Featured`/`Recommended` |
| 3 | **1차 의사결정 5건** | v1 전달분 미회신 — 캠페인 소재 주체·오피포인트 정책(법무/재무)·아동 나이 검증 강도·채널 성장 측정 소스 |

### 개발 선행 조건 (기획과 별개로 확보 필요)

- **`cancel_reason` 수집 흐름** — 취소 시 사유를 기록하지 않으면 대시보드 Cancellation 탭은 데이터 소스 자체가 없다. 취소 UI에 사유 선택 + 과거 취소 건 처리 방침.

## 예정 내용

**1차 회신 반영분**
- 예약생성 OP 편의 기능 상세 스펙 확정 (룸타입 솔팅 · 요금 복사) + **대체 항목 1건 현장 발굴** — 당초 3종이었으나 '더블·트윈 분리 예약' 폐기(2026-07-17). 대체 항목 발굴 또는 KR2 지표를 2건으로 조정할지 결정 필요
- 오피포인트 정책 확정본 (법무/재무 검토 결과 포함)
- 언어팩 5종 콘텐츠 계획 (콘텐츠팀 번역 일정)
- 닷비즈 채널 성장 설계 (신규 고객사 확보 실행안 · 성장 측정 체계)

**2차 회신 반영분**
- 베스트셀러 랭킹 **편집 수단** 설계 (위 표 참조)
- 대시보드 집계 **서버 이관** 설계 + `cancel_reason` 수집 흐름

**미지목 백로그** — [기획서 ② §3.2](../../plan/spec-b-dotbiz-enhancement.md) 참조. 현장 지목 대기 중인 실사이트 차이 6건(요금 이중 슬라이더 · 고정 필터 목록 · Excel 실다운로드 · Voucher/Invoice 인쇄 뷰 · Notice 한국어팩 · 직원/설정 영속화).
