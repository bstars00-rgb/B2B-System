# handoff/ — 기획팀(PD) 전달 패키지

기획 차수별로 폴더를 나눠 관리한다. **N차 기획 = `vN/` 폴더** — 전달 시점의 스냅샷이며, 전달 후에는 내용을 고치지 않고 다음 차수 폴더에서 이어간다(정정이 필요하면 같은 폴더에서 수정하되 문서 상단에 정정 이력 표기).

## 폴더 규칙

```
docs/handoff/
  v1/   ← 1차 기획안 (2026-07-17 전달, Tracy/PD팀)
  v2/   ← 2차 기획 시작 시 여기에 작성 (8월 기획 마무리분)
  v3/   ← 이후 차수도 같은 방식
```

## 각 차수 폴더의 표준 구성

| 파일 | 내용 |
|------|------|
| `README.md` | 패키지 목차 · 전달 대상 · 전달일 · 데모 링크 |
| `proposal-en.md` | 기획안 (영문 — PD팀 전달 기준 언어) |
| `proposal-ko.md` | 기획안 (한글 — 내부 검토용) |
| `dev-spec-en.md` | 개발 명세서 (영문 — 기능 스펙·데이터 계약·QA·코드 맵) |

작업용 상시 문서(기획서 ①·②, 실측 노트)는 [docs/plan/](../plan/)에 그대로 두고, 차수 폴더에서는 링크로 참조한다.
