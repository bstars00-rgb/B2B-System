## 이번 차수 핵심

1. **닷비즈에 없던 신규 화면** — 셀러가 자기 실적(몇 건·얼마·어디로·취소율)을 볼 방법이 전무했다.
2. **모든 수치가 실제 예약에서 파생** — 화면에 박아둔 상수가 없다. 프로토타입 예약 200건과 대조 검증 완료(This Month 43건/¥2,662,009/189박 · This Year 181건/¥9,393,521/685박).
3. **랭킹 → 클릭 → 예약 동선** — 호텔명을 누르면 목적지·호텔이 채워진 Create Booking이 열리고 달력까지 뜬다. 이를 위해 **랭킹을 실제 판매 호텔에서 재구성** — 기존 목록은 320개 중 판매 인벤토리 실재가 **1개**뿐이라 "OhMyHotel 베스트셀러"인데 OhMyHotel에서 못 파는 목록이었다. 표 컬럼도 5→9개(MoM·Hotel Code·Nightly from·Chain Brand).
4. **PD팀 명세 대비 조정 7건** — 특히 ④ Supplier 컬럼·⑤ OP Points 카드는 **경영 판단에 따른 삭제**로, PD팀 명세 문서에서도 제거했다(`Dashboard_Specification_2026-07-17_KR.md` 직접 수정). **통보 사항이며 검토 요청이 아니다.**
5. **PD팀 명세 내부 모순 1건 발견 → 해소** — §4 Account Level 값(`Direct/DIDA/Hotelbeds`)이 PD팀 코드·우리 코드 양쪽과 불일치했다. 회신(① 셀러 자기 예약만 · ⑦ 공급사명 미포함)으로 **필터 자체가 삭제**되어 함께 해소.
6. **한계 1건을 숨기지 않았다** — 예약이 2026-05부터라 Year-End의 2024·2025가 비어 있다. 이력을 지어내면 '실제 예약과 매칭'이 깨지므로 화면에 사유를 표기했다.
