# 02. 프로젝트 지침 (Project Instructions)

> 이 프로젝트에서 반복적으로 지켜온 작업 방식·기준·프롬프트 패턴. OPS 계정에서도 동일하게 적용.

## 1. 프로젝트 기본 지침

1. **클론 우선, 점진 개선**: 전면 리뉴얼 금지. 실제 닷비즈(ohmyhotel.biz) 화면을 1:1로 클론한 뒤 기능을 하나씩 추가한다.
2. **스크린샷 = 명세**: 사용자가 실제 화면 스크린샷을 주면 그것을 명세로 삼아 UI/UX·동작·컬럼·모달 구성을 동일하게 구현한다. 임의 디자인 금지.
3. **실사이트 데이터 반영**: 컬럼명·게시글·호텔코드·계좌·회사정보 등은 실사이트에서 확인된 실제 값을 사용한다(예: 호텔코드 810310, 계좌 Mitsubishi UFJ / 株式会社アティックツアーズ / 5378135).
4. **AI 검색 환각 차단 원칙(축 A)**: 금액·취소조건 숫자는 도구 결과 JSON(요금 카드)에서만 렌더. LLM 텍스트는 Validator로 대조. 예약/결제 도구 미구현으로 구조적 차단. `agent_id`는 서버 주입(사용자 입력 불신). AGENT_USER에게 net/markup 마스킹.
5. **조회 전용(MVP)**: 축 A는 예약 생성/취소/수정/결제 제외. 단, 축 B 클론에서는 실제 포털과 동일하게 예약 생성·취소·인보이스·결제창(UI)까지 Mock으로 재현(실제 결제·전송은 하지 않음, 프로토타입 안내 표기).
6. **매 기능마다 검증→배포**: 코드 변경 시 ① `npm run build`(tsc strict + vite) 통과 ② 브라우저에서 실동작 검증(정상 케이스 + 엣지) ③ 콘솔 에러 0건 확인 ④ 커밋·푸시 ⑤ GitHub Pages 재배포 완료 감시.
7. **한국어로 소통**, 문서·주석·커밋 메시지 모두 한국어 기준(코드 식별자는 영어).

## 2. 반복적으로 사용하는 프롬프트 패턴

- **"OO도 작동하도록 해줘 / OO 클론해줘"** → 해당 실사이트 화면을 스크린샷 기준으로 클론, 데이터·필터·모달까지 동작.
- **"목데이터 넣어줘"** → 초기 시드 데이터를 실사이트 유사하게 채움(예: 예약 7건, 소테츠 호텔들).
- **"UI 동일하게 만들어줘"** → 색상·간격·스크롤 동작(고정 높이 그리드·헤더 고정)까지 실사이트와 맞춤.
- **"빠진 부분 있어"** → 실사이트 스크린샷과 대조해 누락 섹션·컬럼 추가.
- **스크린샷만 첨부** → 그 화면을 그대로 구현하라는 지시로 해석.

## 3. 답변 및 작업 기준

1. **작업 완료 시 반드시**: 무엇을 구현/검증했는지 간결히 보고 + 데모 확인 경로 안내. 배포는 백그라운드 감시로 완료 후 알림.
2. **빌드 실패/버그 발견 시**: 원인 재현 → 수정 → 재검증. (예: vite.config.ts `process.env` 타입 선언 누락으로 CI 실패 → 타입 declare 추가; StaffPage 모달 Row 컴포넌트 내부 정의로 키 입력마다 리마운트되어 포커스 빠지는 버그 → 컴포넌트 외부로 분리.)
3. **URL은 클릭 링크로 제공**(앱 내부 브라우저로 자동 열지 않음). 바탕화면에 바로가기(`B2B 데모 열기.url`) 생성해 둠.
4. **불확실한 정보는 추정 금지 → "확인 필요" 표기**.
5. **안전 규칙 준수**: 실제 카드번호 입력·결제·송금·개인정보 폼 제출은 하지 않는다. 결제창은 UI 재현만(프로토타입 안내 명시). 로그인은 사용자가 직접 수행.
6. **메모리 활용**: 프로젝트 진행·포털 사실은 로컬 메모리에 기록(아래 05 참조).

## 4. 개발 검증 루틴 (고정)

```
cd prototype
npm run build            # tsc --noEmit && vite build (반드시 통과)
# preview_start(prototype-dev) → 브라우저에서 로그인→해당 기능 실동작 확인
# read_console_messages(onlyErrors) → 0건 확인
git add prototype/src && git commit -m "<한국어 요약>" && git push
# GitHub Actions Pages 배포 success 감시 후 완료 보고
```
